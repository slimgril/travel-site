# Content Style v1.0 — 旅行札記

**狀態**：全站預設文風（2026-07-13 採用）  
**適用範圍**：`content/` 下所有旅程 Markdown 內容

---

## 目的

將網站內容由「旅遊景點介紹」正式轉型為「旅行札記」。

讀者應感受到的是旅人在路上的親身經歷，而非翻閱旅遊手冊或百科全書。

---

## 寫作原則

1. **第一人稱旅行者視角** — 以旅途中的親眼所見、親身體驗、當下感受為主。
2. **自然省略主詞** — 不必一直出現「我」，語氣仍屬第一人稱。
3. **體驗優先** — 景點歷史、文化、建築特色僅作背景，篇幅約占 20–30%，不可喧賓奪主。
4. **照片融入情境** — 若照片中出現斌哥本人，文字應自然融入當下情境，而非描述「這是一張合影」。
5. **一圖一故事** — 每一張照片都應有自己的故事，不重複同一段景點介紹。
6. **一圖不拆兩卡（HARD）** — **禁止**把同一張（或同景近重複）照片做成兩張卡片、配兩段不同標題。若同一張照片承載兩層意思（例：憲法牆＋「我是貓王」銘牌），**併在同一張卡、同一段敘事**，把描述寫得更精簡，而不是拆成兩卡。不確定是否同景／重複 → **寧可不上**第二張，也不製造雙描述。
7. **保留結構** — 維持既有 Markdown 區塊（`## 歷史`、`## 古蹟`、`###` 卡片標題與 `![...](path)` 圖片引用），不更動 CSS 或 build 模板。

---

## 避免（❌）

| 類型 | 範例 |
|------|------|
| Wikipedia 式條目 | 「建於 1660 年，為資助新清真寺而建」 |
| 旅遊書導覽 | 「可容納十萬人的賽馬場遺址」 |
| 百科介紹 | 「鄂圖曼帝國六百年統治的中心」 |
| 年代表 / 年代堆砌 | 「自 1963 年就在這裡販售」 |
| 建築規格 | 「近兩萬片藍色磁磚與彩繪玻璃」 |
| 同圖拆兩卡 | 憲法牆一卡＋幾乎同景的「我是貓王」另卡，各寫一段 |

---

## 應使用（✓）

偏好以下動詞與句式，營造現場感：

- 漫步……
- 穿過……
- 抬頭望見……
- 停下腳步……
- 忍不住拍下……
- 留下這張照片……
- 感受到……
- 為今天留下美好的回憶……

## 改寫範例

### 香料市集 Simit 麵包車

**原文（景點介紹）：**

> 市集附近的魯斯坦帕夏（Rüstem Paşa）廣場上，紅色 Simit 麵包車自 1963 年就在這裡賣圈形芝麻麵包——剛出爐的 Simit 外脆內軟，是伊斯坦堡街頭最經典的點心。

**改寫（旅行札記）：**

> 穿過香料市集後，廣場旁這輛販售 Simit 的紅色麵包車格外醒目。剛出爐的芝麻麵包香氣四溢，忍不住停下腳步品嚐，也留下這張充滿伊斯坦堡街頭風情的紀念照。

### 差異說明

| 面向 | 原文 | 改寫 |
|------|------|------|
| 視角 | 第三者介紹 | 旅行者當下體驗 |
| 歷史 | 1963 年開業 | 省略年代，聚焦當下 |
| 結尾 | 定義「經典點心」 | 個人感受與紀念 |

## 增量採用政策

- **全站預設**：自 v1.0 起，所有新增或修改的內容皆遵循本規範。
- **不全面重寫**：既有旅程（山西、西伯利亞大鐵路等）維持原狀，除非該檔案被主動編輯。
- **試點優先**：從 `content/bldh-trio/day01.md`（Baltic Day 1）開始套用，驗證後逐步擴及其他天次與旅程。
- **編輯觸發**：僅在觸及某檔案時才改寫該檔；不因風格升級而批次改寫未動過的內容。
